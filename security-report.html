🔴 致命级漏洞 (5个) 必须修复 ▼
#1 API Key 硬编码在源码中 第45行
智谱AI的API Key直接以明文写在代码里作为默认值。任何人看到源码就能拿到这个Key。
ZHIPU_API_KEY = '8c75d09ce0d04383a813313c74ab7fa8.GXsPMmb8MaUX47e5'
💀 攻击场景
攻击者拿到Key后可以免费调用你的AI接口 → 刷爆配额或产生费用。但当前使用的是免费模型GLM-4-Flash，所以经济损失≈0。
✅ 实际影响：低
因为用的是免费模型且Key已暴露在本次会话中。建议后续轮换Key。
#2 JWT密钥硬编码 + 可预测 第35行
JWT签名密钥是一个固定的字符串，不是随机生成的。
JWT_SECRET = 'chenyu_ai_jwt_secret_key_2026_secure_32bytes'
💀 攻击场景
知道密钥就能伪造任意用户的Token → 冒充管理员/冒充VIP → 但前提是要先拿到源码。
✅ 实际影响：中低
攻击者需要先获取到源码才能伪造Token。公网上源码不可见（只暴露API接口），所以风险降低。
#3 微信登录可无码注册 第717-719行
微信登录接口允许不传code就直接登录，每次创建新账号并获得免费额度。
if not code: openid = f'dev_{int(time.time())}' # 无code也能登录！
💀 攻击场景
写个脚本循环调用 → 每次获得3次免费AI调用 → 白嫖你的API配额。这是目前最可能被利用的漏洞！
✅ 实际影响：中等（最需关注）
有人刷接口的话会消耗智谱AI的免费额度。建议尽快关闭无code登录。
#4 密码用纯SHA-256哈希（无盐值）第201-203行
密码只用简单的SHA-256加密，没有加随机盐值(salt)。
return hashlib.sha256(password.encode('utf-8')).hexdigest()
💀 攻击场景
如果数据库文件泄露 → 彩虹表秒破解所有密码 → 但数据库在你本地电脑上。
✅ 实际影响：低
SQLite数据库在你本地电脑上，不在公网。外部无法访问。
#5 管理员弱密码 admin/admin123 第172-174行
默认管理员密码是常见的弱密码，且明文写在注释里。
💀 攻击场景
猜到admin密码 → 登录后台 → 操作所有用户数据/订单。但管理员接口需要Token验证。
✅ 实际影响：低
需要先有合法账号的Token才能调管理员接口。改掉默认密码即可消除风险。
🟠 高危漏洞 (7个) 建议修复 ▶
